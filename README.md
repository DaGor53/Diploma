В данном репозитории содержатся следующие файлы:
Sound_Model.ipynb - содержит ноутбук с кодом для обучения CNN (свёрточной нейронной сети) на датасете, состоящим из трёх mel-спектрограмм на канал сопоставленных к углу определения на объект.
Visual_Model.ipynb - содержит ноутбук с кодом для обучения модели YOLO для определения на специализированном собранном датасете наличие объекта (класс 0, БПЛА) в кадре, а также последующим определением угла на цель от 0 до 360.
ch_num.py - утилита, определяющая число каналов в .wav файле для проверки корректности записанных аудиоданных
class.py - утилита для обработки разрозненных датасетов YOLO, переименовывающая все найденные классы в 0. Датасет предварительно должен быть очищен от лишних изображений.
h264tomp4.py - утилита, конвертирующая .h264 файлы, записанные на Raspberry Pi, в формат .mp4.
mp4_cut.py - утилита для обработки записанных на установку видео-файлов, обрезает видео. Для корректной работы необходимо задать начальное значение минут и секунд, а также конечное значение минут и секунд. Края обрезаются, остаётся середина.
rename.py - утилита для обработки датасетов, унифицирует имя файлов.
converter.py - утилита, склеивающая вместе записанные файлы .h264 и .wav файлы в цельное видео. 
test14.py - скрипт, разворачивающийся на Raspberry Pi 4B. 


Далее идёт детальное описание работы test14.py и его применение на практике. 
Систему предварительно необходимо включить путём подсоединения её к аккумулятору через разъём питания. После этого необходимо подождать от 30-40 секунд пока система Raspberry Pi OS загрузится.
После этого, в том случае, если этого не было сделано ранее, скрипт необходимо по протоколу ssh загрузить на систему с помощью компьютера или терминального приложения для телефона. Пароль, имя устройства и сеть выбираются в индивидуальном порядке.
Затем скрипт запускается через терминал. 
Дальнейшее управление производится от установленной на системе кнопки. Единичное нажатие: начало записи видео-данных с камеры, а также подсоединённых микрофонов. Аудио записывается 
отдельно от видео, а затем оба файла архивируются с меткой даты и сохраняются на SD-карту. Повторное короткое нажатие кнопки прерывает запись и формирует архив. Длинное нажатие прерывает запись, формирует архив и совершает выход из цикла программы, делая систему вновь доступной для подключения по ssh.
