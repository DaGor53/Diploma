В данном репозитории содержатся следующие файлы:

1. Sound_Model.ipynb - содержит ноутбук с кодом для обучения CNN (свёрточной нейронной сети) на датасете, состоящим из трёх mel-спектрограмм на канал сопоставленных к углу определения на объект.
2. Visual_Model.ipynb - содержит ноутбук с кодом для обучения модели YOLO для определения на специализированном собранном датасете наличие объекта (класс 0, БПЛА) в кадре, а также последующим определением угла на цель от 0 до 360.
3. ch_num.py - утилита, определяющая число каналов в .wav файле для проверки корректности записанных аудиоданных
4. class.py - утилита для обработки разрозненных датасетов YOLO, переименовывающая все найденные классы в 0. Датасет предварительно должен быть очищен от лишних изображений.
5. h264tomp4.py - утилита, конвертирующая .h264 файлы, записанные на Raspberry Pi, в формат .mp4.
6. mp4_cut.py - утилита для обработки записанных на установку видео-файлов, обрезает видео. Для корректной работы необходимо задать начальное значение минут и секунд, а также конечное значение минут и секунд. Края обрезаются, остаётся середина.
7. rename.py - утилита для обработки датасетов, унифицирует имя файлов.
8. converter.py - утилита, склеивающая вместе записанные файлы .h264 и .wav файлы в цельное видео. 
9. test14.py - скрипт, разворачивающийся на Raspberry Pi 4B.
10. finder2.py - файл, запускаемый с компьютера. Использует обученную модель машинного обучения YOLO для покадрового просматривания записанных на Raspberry Pi 4B видео для целей формирования датасета для обучения аудио-модели. Модель должна определять факт наличия и угловое положение объекта в кадре с камеры. Записанные видео должны быть формата .mp4 и лежать в одной папке. Аудиоданные для каждого кадра берутся как момент кадра +- секунда. На выходе создаются три mel-спектрограммы на единицу данных, а также сопоставляет ему пару с json-файлом, в котором хранятся данные: имя родительского видео, номер кадра, угол на объект, границы бокса и уверенность модели.

Также хранятся следующие файлы моделей:
1. audio1.tflite - обученная аудио-модель конвертированная в формат .tflite
2. audio2.tflite - вторая версия обученной аудио-модели, конвертированной в формат .tflite
3. best1_int8_320.tflite - первая обученная видео-модель, конвертированна я в .tflite
4. best2_int8_320.tflite - вторая обученная видео-модель, конвертированна я в .tflite
5. best3_int8_320.tflite - третья обученная видео-модель, конвертированна я в .tflite
6. last.pt - третья обученная видео-модель, рассчитанная на запуск скриптом finder2.py на компьютере 

Далее идёт детальное описание работы test14.py и его применение на практике. 

Систему предварительно необходимо включить путём подсоединения её к аккумулятору через разъём питания. После этого необходимо подождать от 30-40 секунд пока система Raspberry Pi OS загрузится.
После этого, в том случае, если этого не было сделано ранее, скрипт необходимо по протоколу ssh загрузить на систему с помощью компьютера или терминального приложения для телефона. Пароль, имя устройства и сеть выбираются в индивидуальном порядке.
Затем скрипт запускается через терминал. 
Дальнейшее управление производится от установленной на системе кнопки. Единичное нажатие: начало записи видео-данных с камеры, а также подсоединённых микрофонов. Аудио записывается 
отдельно от видео, а затем оба файла архивируются с меткой даты и сохраняются на SD-карту. Повторное короткое нажатие кнопки прерывает запись и формирует архив. Длинное нажатие прерывает запись, формирует архив и совершает выход из цикла программы, делая систему вновь доступной для подключения по ssh.
